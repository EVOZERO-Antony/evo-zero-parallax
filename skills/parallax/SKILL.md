# Parallax

Treat Parallax as a decision instrument, not a verdict generator. Preserve human ownership of the final decision.

## Model the instrument

Map astronomical parallax to reasoning:

- **Target:** the claim or verdict under test.
- **Fixed reference:** a validated fixpoint that should remain true as the situation and altitude change.
- **Baseline:** separation created by moving upward in abstraction, not by relabeling the same viewpoint.
- **Reading:** how contingent facts move relative to the fixpoint.

Keep three controls independent:

1. **Altitude:** generate real angular separation. Use at least three levels, such as raw situation, surrounding
   structure, non-copyable ownership, and the meta-question.
2. **View:** select a disciplinary seat such as builder, buyer, operator, adversary, or outsider. Hold it fixed by
   default so altitude and attack remain observable.
3. **Attack:** run both directions at every altitude.
   - **Bull:** build the strongest honest case for the target by trying to disconfirm the threat.
   - **Bear:** build the strongest honest case against the target by trying to disconfirm the opportunity.

Do not substitute bear-only red-teaming for Parallax.

## Run the procedure

### Phase 0 — Frame

State the target precisely. Define a decision scale such as `GO / lean-GO / HOLD / lean-NO / NO`. Gather and verify
material facts. Separate fact verification from verdict testing.

### Phase 1 — Validate the anchor

1. Break the candidate fixpoint into atomic statements.
2. For each atom, report `truly_fixed: yes/no` and explain whether it stays true when both the situation and altitude
   change.
3. Reject atoms that move along the deciding axis.
4. If the candidate moves, climb one altitude and search for the higher invariant.
5. Distinguish truth from advantage: a genuine fixpoint may be strategically inert.

Stop and report the boundary if no genuine invariant can be established. The instrument cannot measure without one.

### Phase 2 — Build the grid

Cross at least three altitudes with `{bull, bear}` to create at least six cells. Hold view fixed unless the user asks
to vary it. Run cells independently where possible; otherwise reason through them sequentially without allowing one
cell's conclusion to dictate another.

### Phase 3 — Commit each cell

For every cell, report:

- altitude and attack;
- apparent reading on the decision scale;
- contingent facts that moved at this altitude;
- whether the fixpoint still appears fixed;
- concise reasoning.

Make bull cells genuinely bullish and bear cells genuinely bearish. Do not disguise the preferred conclusion as both.

### Phase 4 — Synthesize and criticize

Run two distinct passes over the completed grid:

1. **Synthesis:** identify convergence across opposing attacks and altitudes; report the center and spread as a
   confidence band; classify the pattern as bracketed, drift, or unresolved.
2. **Adversarial critic:** test whether bulls are bears in disguise, whether lenses share a hidden offset, whether the
   fixpoint is load-bearing, and whether residual drift runs in either direction.

### Phase 5 — Return authority

Return the reading, uncertainty, deciding axis, and next reality test to the human decision-maker. Do not present the
instrument's output as automatic authority. Record only assumptions and findings that survived the critic.

## Interpret the reading

- **Bracketed:** bull and bear readings oppose each other around a stable center. Treat the center as the current
  reading and the spread as uncertainty.
- **Drift:** cells lean in one direction despite attack and altitude. Treat this as a possible instrument failure;
  re-check anchor fixity, altitude separation, and attack integrity before trusting the conclusion.
- **Unresolved:** all cells remain conditional because a deciding axis cannot be observed from analysis alone. Name
  that axis and propose a bounded contact-with-reality test.

Use the optimum altitude, not the maximum altitude. Stop climbing when the target begins to dissolve into generality.

## Output contract

Return sections in this order:

1. `Target and scale`
2. `Validated fixpoint` with atom-level self-test
3. `Parallax grid`
4. `Synthesis`
5. `Adversarial critic`
6. `Reading, uncertainty, and reality test`
7. `Human decision boundary`
