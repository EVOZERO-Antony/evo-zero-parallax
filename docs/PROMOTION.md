# Parallax launch and promotion kit

## GitHub description

`An EVO ZERO decision instrument for pressure-testing verdicts across abstraction levels and mirrored bull/bear attacks.`

## One-line pitch

Parallax catches a decision instrument drifting before you trust its verdict.

## Short launch post

Most decision tools ask for more perspectives. Parallax asks a harder question: did those perspectives create a real
angle, or did they merely pan across the same hidden bias?

Today I am releasing Parallax v0.1.0, a standalone AI skill descended from EVO ZERO. It validates a fixed anchor, runs
bull and bear attacks at multiple levels of abstraction, and then audits its own reading before returning the decision
to the human.

Use it when the answer feels obvious — and the cost of being wrong is not.

`Run parallax on this decision.`

## Demo format

Publish one complete, anonymized case with:

1. the seductive obvious verdict;
2. the rejected false anchor;
3. one fact that changes meaning across altitudes;
4. the bull/bear bracket;
5. the final reality test;
6. what the human decided afterward.

This demonstrates the mechanism better than a feature list.

## Credibility assets to add over time

- `examples/`: three bounded cases with inputs and full outputs;
- `evals/`: failure-focused tests for fake bulls, false anchors, and coplanar altitudes;
- a short terminal or screen recording;
- release notes and dated tags;
- independent case reports that include failures, not only wins;
- a public EVO ZERO parent page linked from the lineage document.

## Promotion sequence

1. Publish repository and signed/tagged v0.1.0 release.
2. Post the worked example, not just the announcement.
3. Invite five practitioners to test the same prompt on real decisions.
4. Convert recurring failures into public evals and issue labels.
5. Release v0.1.1 with evidence of what changed.
6. Write the EVO ZERO lineage article only after readers can use Parallax standalone.

## Claims discipline

Lead with observable mechanics and examples. Avoid guarantees, invented benchmarks, or vague "scientifically proven"
language. Public timestamp, citation metadata, tagged releases, and a precise lineage statement do more for authorship
than inflated novelty claims.
