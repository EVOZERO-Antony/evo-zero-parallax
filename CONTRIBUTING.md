# Contributing to Parallax

Contributions are welcome when they make the instrument clearer, more falsifiable, or easier to run without weakening
its controls.

## Before opening a pull request

1. Describe the failure mode or concrete use case.
2. Explain which invariant is preserved or improved.
3. Include a before/after example when behavior changes.
4. Run the skill validator.
5. Avoid claims that Parallax guarantees truth, eliminates bias, or replaces human judgment.

## Non-negotiable controls

- Validate a fixpoint before measurement.
- Use at least three genuine abstraction levels.
- Run both bull and bear at every altitude.
- Keep synthesis and adversarial criticism distinct.
- Return final authority and consequence ownership to the human.

Use GitHub Issues for proposed changes. Security-sensitive reports should follow [`SECURITY.md`](SECURITY.md).
